Практична робота №3

Дисципліна: Проектний практикум

Тема: Вибір технології, мови та середовища програмування

Варіант виконання: Вебзастосунок для онлайн-торгівлі

1. Опис задачі та визначення вимог до системи

Задача:
Розробка вебзастосунку для онлайн-торгівлі, який забезпечує можливість перегляду каталогу товарів, додавання товарів до кошика, оформлення замовлень та проведення безпечних платежів. Система повинна бути здатна обслуговувати велику кількість користувачів, зокрема під час акцій та розпродажів.

Основні вимоги до системи:

  • Підтримка високої відвідуваності:
    Забезпечення безперебійної роботи системи навіть при високій кількості одночасних користувачів, завдяки високопродуктивній серверній архітектурі.

  • Безпека:
    Захищений процес оплати та обробки даних користувачів із використанням сучасних методів шифрування та протоколів безпеки (наприклад, HTTPS).

  • Адаптивний інтерфейс:
    Зручний та інтуїтивно зрозумілий інтерфейс, який однаково добре працюватиме як на десктопних, так і на мобільних пристроях.

  • Інтеграція з інвентарною системою:
    Реальний час оновлення інформації про наявність товарів, що дозволяє користувачам бачити актуальні дані про запаси.

2. Аналіз та вибір технологій, мови програмування та середовища

Мови програмування:

  • JavaScript:
    Використовується для розробки як клієнтської, так і серверної частин застосунку. Є перевагою, адже дозволяє використовувати єдину мову в усьому проєкті.

  • Python:
    Може бути використаний для швидкої розробки прототипів та аналітики, але менш оптимальний для високонавантажених систем.

  • Java:
    Надійний вибір для масштабованих систем, однак у нашому випадку ми віддаємо перевагу єдиній екосистемі JavaScript.

Технології та середовища:

  • Серверна частина:
    Node.js – середовище виконання JavaScript на сервері, яке забезпечує неблокуючу архітектуру та високу швидкість обробки паралельних запитів.

  • Клієнтська частина:
    React – сучасний фреймворк для створення динамічних та адаптивних інтерфейсів, що дозволяє швидко оновлювати лише необхідні компоненти сторінки.

  • Бази даних:

    • MySQL: Реляційна база даних для зберігання структурованої інформації (каталог товарів, замовлення, дані користувачів).
    • MongoDB: Нереляційна база даних для роботи з гнучкими та неструктурованими даними (логування, сесії користувачів, інтеграція з іншими сервісами).

3. Обґрунтування вибору технологій

Серверна частина – Node.js:

  • Переваги:

    • Неблокуюча, асинхронна обробка запитів, що забезпечує високу продуктивність.
    • Простота інтеграції з фронтендом за рахунок використання однієї мови – JavaScript.
    • Легке масштабування системи, що є критичним для вебзастосунку з високою відвідуваністю.

  • Недоліки:

    • Обмежена ефективність при виконанні складних числових обчислень, проте для роботи з API та обробкою запитів цей недолік мінімальний.

Клієнтська частина – React:

  • Переваги:

    • Компонентний підхід дозволяє створювати повторно використовувані елементи інтерфейсу.
    • Забезпечує часткове оновлення сторінки без повного перезавантаження, що сприяє швидкому та зручному користуванню.
    • Легко адаптується під різні розміри екранів, що важливо для мобільних пристроїв.

  • Недоліки:

    • Для дуже масштабних проєктів може виникнути потреба у використанні додаткових інструментів (наприклад, Redux) для управління станом.

Бази даних:

  • MySQL:

    • Підходить для зберігання структурованих даних з фіксованою схемою.
    • Забезпечує підтримку складних запитів та транзакцій, що важливо для обліку замовлень та даних товарів.

  • MongoDB:

    • Ідеальна для зберігання неструктурованих даних або даних, що часто змінюються.
    • Легко масштабується та забезпечує високу швидкість роботи з великими обсягами даних.

4. Приклади використання обраних технологій

  • Node.js:
    Обробка запитів користувачів здійснюється за допомогою Express.js – легкого фреймворку, який дозволяє швидко налаштовувати маршрути та обробники запитів. Це забезпечує можливість ефективної обробки великої кількості одночасних підключень.

  • React:
    Використовується для створення інтерактивних компонентів інтерфейсу, наприклад, динамічного кошика покупок. Завдяки Virtual DOM, оновлення інтерфейсу відбувається швидко, що сприяє кращому користувацькому досвіду.

  • MySQL та MongoDB:

    • MySQL застосовується для зберігання основних даних, таких як інформація про товари, замовлення та користувачів, забезпечуючи надійність та цілісність даних.
    • MongoDB використовується для зберігання сесій користувачів, логів активності та інших гнучких даних, що потребують швидкого доступу та змін.

5. Відповіді на контрольні запитання

  1. Які фактори впливають на вибір технології та мови програмування для конкретного проєкту?
    Факторами є тип задачі, вимоги до продуктивності, масштабованість, безпека, сумісність із іншими системами, наявність бібліотек та фреймворків, а також експертиза розробників.

  2. Які вимоги до системи важливо врахувати під час планування вебзастосунку?
    Важливо врахувати підтримку високої відвідуваності, безпеку даних, адаптивність інтерфейсу для різних пристроїв та інтеграцію з іншими системами (наприклад, інвентарною системою).

  3. Які основні відмінності між реляційними та нереляційними базами даних, і як ці відмінності впливають на вибір бази даних?
    Реляційні бази даних (наприклад, MySQL) використовують структуровані таблиці з фіксованою схемою, підтримують складні запити та транзакції, що добре підходить для структурованих даних. Нереляційні бази даних (наприклад, MongoDB) є схематично вільними, дозволяють зберігати неструктуровані дані та легко масштабуються, що корисно для даних, що часто змінюються або не мають жорсткої структури.

  4. Чому Node.js є популярним вибором для серверної частини вебзастосунків?
    Node.js використовує неблокуючу, асинхронну модель вводу/виводу, що забезпечує високу продуктивність та здатність обробляти велику кількість одночасних запитів, що є критично важливим для високонавантажених систем.

  5. Які переваги використання React для розробки клієнтської частини, особливо для мобільних інтерфейсів?
    React дозволяє створювати компонентний та адаптивний інтерфейс, оптимізувати оновлення сторінки за допомогою Virtual DOM і забезпечувати швидкий та зручний досвід користувача як на десктопах, так і на мобільних пристроях.

  6. Як можна обґрунтувати вибір певної технології для проєкту?
    Вибір технології обґрунтовується відповідністю її можливостей вимогам проєкту, такими як продуктивність, масштабованість, безпека, доступність бібліотек та фреймворків, а також досвідом розробницької команди.

  7. Як масштабованість та продуктивність обраної технології можуть впливати на ефективність системи?
    Масштабованість дозволяє системі обробляти зростаючі навантаження без втрати швидкодії, а висока продуктивність забезпечує швидку реакцію на запити, що критично для підтримки гарного користувацького досвіду.

  8. Які обмеження можуть виникнути при використанні конкретних технологій або баз даних?
    Обмеження можуть включати проблеми з обробкою складних обчислень (наприклад, Node.js не найкращий для числово-інтенсивних задач), труднощі з управлінням великими обсягами даних або інтеграцією між різними системами, а також потребу в додаткових інструментах для управління станом у масштабних проєктах.

  9. Чому важливо враховувати сумісність обраних технологій між собою?
    Сумісність технологій забезпечує безперебійну інтеграцію різних компонентів системи, знижує ризики виникнення помилок під час взаємодії та дозволяє створити ефективну та стабільну інфраструктуру.

6. Висновок

У результаті аналізу вимог до вебзастосунку для онлайн-торгівлі було обрано використання Node.js для серверної частини та React для клієнтської частини. Цей вибір дозволяє забезпечити високу продуктивність, масштабованість та адаптивність системи, а також сприяє швидкій та ефективній розробці завдяки використанню єдиної екосистеми JavaScript. Використання MySQL та MongoDB дозволить оптимально організувати зберігання як структурованих, так і неструктурованих даних, що є важливим для підтримки стабільності роботи застосунку.
