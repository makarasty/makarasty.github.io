- -p protocol — інформація лише для вказаного протоколу (TCP/UDP).

name		 Перетворює DNS-ім’я у IP-адресу та навпаки (допомагає перевірити роботу DNS).

7 net use /
net view		 (Без параметрів або вказати ресурс) - net use — підключення або відключення мережевих дисків/папок.

8 ping назва_ПК або IP
-t
-a
-r
-i		 - ping назва_ПК/IP — перевіряє доступність вузла.

- -r — записує маршрут для заданої кількості переходів.

-p
-w		 - -d — не визначає імена вузлів (тільки IP-адреси).

Завдання 2

Визначити мережеву назву (hostname) комп’ютера. Повну відповідь записати у звіті.

  1. Відкриваю Командний рядок:

    Win + R → cmd → Enter

  2. Виконую команду:

    hostname

  3. Система виводить назву комп’ютера:

    MKY

  4. Відповідь:

    Мережеве ім’я комп’ютера: MKY

Завдання 3

*Визначити значення поточних параметрів мережі (IP-адреса, маска, шлюз, DNS). Записати результат у Таблицю 2.

  1. У Командному рядку виконую команду:

    ipconfig /all

  2. Знаходжу дані про активний мережевий адаптер (Ethernet або Wi-Fi), зокрема:

    • IPv4 Address (IP-адреса)

    • Subnet Mask (Маска мережі)

    • Default Gateway (Основний шлюз)

    • DNS Servers (Адреси DNS-серверів)

Таблиця 2. Поточні параметри мережі

Параметр Значення
IP-адреса (IPv4) 192.168.1.101
Маска мережі 255.255.255.0
Основний шлюз (Gateway) 192.168.1.1
DNS-сервер 8.8.8.8

Відповідь:

  • IP-адреса: 192.168.1.101
  • Маска: 255.255.255.0
  • Шлюз: 192.168.1.1
  • DNS: 8.8.8.8
